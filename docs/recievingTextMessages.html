Receiving Text Messages

Sending text messages with RapidAPI is pretty simple, but imagine if you could also receive and act upon text messages from your users? Well now you can, with a few basic steps:


Create a phone number

When sending a text message, you can send it from our default number, but when receiving messages, each project has to have it’s own number (as otherwise we won’t be able to know which project to route the message to). To create a phone number, head over to the Overview tab in the project page. There, you will see the Phone numbers pane, where you can see existing numbers, create new ones and delete old ones.

When creating a new number, you will have to create an endpoint which will be called when the number receives a message. You can either let the system create a new endpoint for you (and just choose code/blocks), or point to an existing endpoint (note that this must be a POST endpoint). Note that you can have more than one number pointing to the same endpoint, as with each request, the number to which the message was sent will be passed along.

Another option is to use the createNumber block in the delivery package to create numbers dynamically from endpoints.


React to a text message

On the endpoint that is hooked to the phone number, you can perform any tasks you can think of (interact with databases, send emails etc…).

You can use sessions to save information related to each number. Each phone number has it’s own session, so you can save data there related to this number (for example, you can send a message and ask the user for his name, and then store it in the session so everytime he sends a message, you will now his name).

All the data related to the text message can be found in the body of the request to the endpoint. Here is the data included:


Information about receiving number:
 

  • To - the number to which the message was sent (if you have more than one number hooked to the same endpoint).
  • ToCountry - the country to which the message was sent
  • ToState - the state to which the message was sent
  • ToCity - the city to which the message was sent


Information about sending number:
 

  • From - the number that sent the message
  • FromCountry - the country from which the message was sent
  • FromState - the state from which the message was sent
  • FromCity - the city from which the message was sent


Information about the message:
 

  • MessageSid - a unique identifier for the message
  • Body - the message itself


Send a message from that number

You may have notice that when you send text messages using the delivery.sendSMS block, they are sent from an arbitrary number. If you have added a number to your project and you are receiving text messages on it, you probably want to send messages from the same number.

For that exact purpose we have the delivery.sendSMSFrom block that let’s you specify the number you want the message to be sent from. In the from field, just put the number you have added to your project (no spaces / dashes, starting with the +. Like that +15104882333). The to and message field will be used as usual.
